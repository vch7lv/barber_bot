# Требования к Telegram-боту для барбера

## Кратко

Бот для барбера и клиентов: клиенты записываются на визит (одна или несколько услуг), видят слоты и прайс, получают напоминания; барберы управляют расписанием, прайсом, записями, рассылкой, банами и видят статистику. Часовой пояс МСК, до ~100 клиентов, ежедневные бэкапы, обязательное логирование.

---

## 1. Вступление

Telegram-бот предназначен для барбера и его клиентов: запись на визиты, управление расписанием и прайс-листом, рассылка новостей, общение барбера с клиентами через бота.

---

## 2. Роли

| Роль | Описание |
|------|----------|
| **Клиент** | Пользователь Telegram, не входящий в список барберов. Забаненный клиент не может записываться и пользоваться основным функционалом. |
| **Барбер** | Барбер = админ. Полный доступ к панели барбера. Барберов может быть несколько (разные Telegram-аккаунты). Список Telegram User ID задаётся при запуске. |

---

## 3. Глоссарий

| Термин | Значение |
|--------|----------|
| **Визит / запись** | Одна запись клиента на дату и время начала; может включать несколько услуг. |
| **Слот** | Непрерывный интервал в расписании барбера. Длительность слота записи = сумма длительностей выбранных услуг. |
| **Услуга** | Элемент прайс-листа: название, цена, длительность (минуты). |
| **МСК** | Часовой пояс Europe/Moscow; все времена в боте — по Москве. |

---

## 4. Функциональные требования

### 4.1. Клиент

| ID | Требование | Описание |
|----|------------|----------|
| K1 | Запись на визит | Выбор одной или нескольких услуг из прайс-листа, затем дата и время начала. Длительность слота = сумма длительностей выбранных услуг. Показываются только слоты, куда помещается весь блок без наложений с другими записями. |
| K2 | Отмена записи | Клиент может отменить свою будущую запись в любой момент до времени визита. |
| K3 | Просмотр свободных слотов | Доступные дни и время начала слотов. Слот свободен, если помещается непрерывный интервал длительностью = сумма длительностей выбранных услуг (с учётом графика и существующих записей). Время — МСК. |
| K4 | Напоминание | В день записи — одно напоминание за **2 часа** до времени визита (МСК). |
| K5 | Просмотр прайс-листа | По запросу показывается актуальный прайс: услуги, цены, длительность. |
| K6 | Просмотр своих записей | Клиент видит свои предстоящие записи: дата, время, список услуг. |
| K7 | Сообщения от барбера | Клиент может получать рассылку новостей от барбера (связь с В3)

### 4.2. Барбер

| ID | Требование | Описание |
|----|------------|----------|
| B1 | Просмотр записей | Список записей с фильтрацией: по датам, по клиентам; день / неделя / месяц. |
| B2 | График и слоты | Рабочие дни и временные промежутки (например Пн–Пт 10:00–20:00). У каждой услуги в прайсе — длительность (мин). Длительность записи = сумма длительностей услуг. Шаг начала записи настраивается (напр. 15 или 30 мин). Свободный слот — непрерывный интервал нужной длины без пересечений с другими записями. Время — МСК. |
| B3 | Рассылка новостей | Отправка сообщения (текст, при необходимости медиа) всем клиентам. |
| B4 | Прайс-лист | Редактирование прайса: название услуги, цена, длительность (мин). Используется для расчёта слота и отображается клиентам (K5). |
| B5 | Отмена записи | Барбер может отменить любую запись (с указанием причины и уведомлением клиента при необходимости). |
| B6 | Список клиентов | Просмотр клиентов: имя, username, Telegram ID, контакт, история/количество записей. |
| B7 | Бан клиента | Бан по Telegram ID. Забаненный не может записываться, отменять через бота, получать рассылки; при попытке — сообщение о блокировке. Возможность разбана. |
| B8 | Статистика | Выручка за период, количество визитов. Разбивка по услугам (сколько раз заказана, сумма), по периодам (день/неделя/месяц), топ клиентов по визитам. В одной записи может быть несколько услуг. |

### 4.3. Конфигурация и администрирование

| ID | Требование | Описание |
|----|------------|----------|
| A1 | Параметры при запуске | Telegram-токен бота, список Telegram User ID барберов; остальное — env/конфиг/флаги. |
| A2 | Резервное копирование | Ежедневный бэкап вечером (время настраивается). Формат — удобный для БД (дамп SQLite/PostgreSQL или SQL). Восстановление — загрузкой дампа. Формат и способ восстановления документируются. Рекомендуется хранить копии вне основного сервера. |
| A3 | Docker Compose | Предоставляется **docker-compose** для поднятия сервера бота: приложение и при необходимости БД/зависимости. Запуск окружения одной командой (например `docker-compose up`). Параметры (токен, ID барберов и т.д.) передаются через переменные окружения или env-файл. |

## 5. Нефункциональные требования

| ID | Требование | Описание |
|----|------------|----------|
| N1 | Масштаб | До ~200 уникальных клиентов; простая архитектура без шардирования. |
| N2 | Доступность | Корректная работа после перезапусков; персистентное хранилище; при возможности — повтор при временных ошибках Telegram API. |
| N3 | Безопасность | Доступ к панели барбера только у барберов; клиенты не имеют доступа к расписанию, рассылкам, списку других клиентов. |
| N4 | Локализация | Язык интерфейса зафиксирован - русский
| N5 | Часовой пояс | Все времена — **МСК (Europe/Moscow)**. |
| N6 | Логирование | Обязательная запись событий: создание/отмена записи, рассылка новостей, ошибки приложения и внешних вызовов (Telegram API и т.д.) — для диагностики и аудита. |
| N7 | KISS | Код должен быть максимально простым и лаконичным (принцип «Keep It Simple, Stupid»). |

---

## 6. Принятые решения (справочно)

| Тема | Решение |
|------|---------|
| Роли | Барбер = админ; барберов может быть несколько; список ID при запуске. |
| Слоты | Настраиваются барбером (длительность услуги, шаг расписания). |
| Отмена клиентом | В любой момент до времени визита. |
| Напоминание | За 2 часа до визита. |
| Часовой пояс | МСК. |
| Бэкапы | Ежедневно вечером; формат для БД (дамп/SQL). |
| Статистика | Выручка, кол-во заказов, по услугам и периодам. |
| Бан | По Telegram ID; разбан возможен. |
| Несколько услуг | В одной записи — несколько услуг; слот = сумма длительностей. |
| Стиль кода | KISS: максимальная простота и лаконичность. |
